꿀팁 보기

사용자는 다양한 브랜드를 조회 순, 최신 순, 빨대 순, 댓글 순 등으로 정렬하여 조회가 가능하며 각 팁들은 저 마다의 커버 스타일과 스크롤 방식을 가집니다.

꿀팁 등록

사용자는 원하는 브랜드를 선택하여 자신이 원하는 방식과 자신이 원하는 형태의 이미지로 제작 가능하며 팁을 등록하여 댓글, 빨대, 좋아요 등의 커뮤니티 서비스를 누릴 수 있습니다.

이제 우리 서브웨이 갈 때 마다 꿀조합 검색하는거 그만해요 ㅠㅠ 이제 빨대하세요!! 사용자는 빨대한 팁을 빠르게 단 두 번의 터치로 확인 할 수 있습니다.

꿀팁 보기

- 최신 순, 조회 순, 댓글 순, 빨대 순으로 정렬이 가능합니다.
- 꿀팁은 상하, 좌우 스크롤이 가능합니다.
- 꿀팁 내에 댓글과 빨대하기(북마크)가 가능합니다.

꿀팁 등록

- 제목과 브랜드를 선택하여 꿀팁을 등록합니다.
- 사용자는 자신이 원하는 사진과 커버 타입, 스크롤 타입등을 지정 할 수 있습니다.

마이페이지

- 자신의 페이지에서 게시한 팁과 꿀팁 수, 빨대 수, 좋아요 수를 확인 할 수 있습니다.
- 자신의 페이지에서 팁을 삭제 할 수 있습니다.
- 다른 사용자의 정보 또한 확인 가능합니다.

댓글 달기

- 원하는 팁에 댓글을 등록 할 수 있습니다.
- 댓글 최상단에는 베스트 댓글이 자리합니다.
- 자신이 올린 댓글을 삭제 할 수 있습니다.

알림 기능

- 사용자는 댓글, 빨대, 좋아요에 대한 알림을 수신 할 수 있습니다.
- 알림은 확인여부에 따라 다른 색으로 표시됩니다.
- 알림을 클릭하여 삭제하거나 알림에 따라 팁, 댓글 화면으로 이동합니다.
- 알림의 수에 따라 앱 아이콘에 뱃지가 표시됩니다.

검색 기능

- 원하는 팁을 검색 할 수 있습니다.
- 검색 기록은 언제든 삭제가능하며
- 자동완성 기능을 지원합니다.

개발 일기

Issues and trouble-shooting stories

  • 첫 번째

    Aws s3 / multer

    step.1
    s3 storage 에 node.js multer 모듈을 사용하여 다중 이미지를 업로드 하는 것이 목적이었다.
    step.2
    upload.single()로 하나의 이미지를 올려본 것이 서비스 이용 경험의 전부인지라 많이 헤매었던 것 같다. 파일의 key값이 unique 하지 않아 array() 형태로 받는 것 또한 불가능한 것이 문제였다.
    step.3
    multer 내 upload.any() 의 사용으로 키값의 영향받지 않고 이미지 배열을 받아와 해결 할 수 있었다.

  • 두 번째

    Out Of Memory

    step.1
    사용자로부터 이미지를 받아 업로드하는 것이 목적이었다.
    step.2
    원인을 찾기 위해 allocation tracking을 수행한 결과 bitmap이 대부분의 메모리를 차지하고 있었다.
    step.3
    이슈 발생의 원인은 이미지를 resizing 하기 위해 사용한 bitmap 과 multi-part 시 사용한 stream에 대한 참조를 해제하지 않아 메모리를 잡아먹던 것이었다. 객체에 대한 참조를 끊어 gc가 가능하도록하여 해결 할 수 있었다.

  • 마지막

    SparseArray

    step.1
    하나의 프레그먼트를 사용하는 뷰페이저 각 페이지에 이미지와 내용을 가져오는게 목적이었다.
    step.2
    뷰페이저 좌우와 현재 보여지는 뷰만을 로드하기 때문에 고민중이었다 물론 pageOffscreenPageLimit() 함수를 통해 로드되는 페이지를 늘릴 수 있으나 동일한 프레그먼트를 사용 중이기에 이 또한 옳지 않은 방법 이었다.
    step.3
    마지막으로 택하게된 것이 바로 Map 의 성능을 높힌 SparseArray 였다. instantiateItem가 호출될 때 position 을 key로하여 프레그먼트를 담는다. 반대로 destroyItem가 호출될 때 해당 position의 값을 제거한다. 프레그먼트를 반환하는 함수를 작성하여 원하는 position의 페이지를 접근하는 것이 가능하였다.
